# 🚀 Quantum-Omni-Tool: Physics-Driven Digital Twin (V2.0)

## Overview
This repository contains a **Digital Twin simulation pipeline** built for the **NVIDIA Omniverse** platform using **OpenUSD** and **Python**. 

The project moves beyond basic visualization to simulate **Quantum Decoherence**—the process by which quantum information is lost due to environmental interference. By leveraging the NVIDIA PhysX engine, this tool links gravitational potential energy (fall height) to the probability of quantum state collapse.

## 🛠 Key Features
* **Procedural Scene Generation:** Dynamically generates a 100-qubit grid using the `pxr` Universal Scene Description (USD) framework.
* **Deterministic Physics Logic:** Implements a custom algorithm where the physical stress of an impact determines the probability of a qubit staying in a stable (**Alpha**) state or decohering into an error (**Beta**) state.
* **Automated Data Acquisition:** Includes a scientific data logger that exports real-time simulation results to CSV for empirical analysis.
* **Modular Architecture:** Built as a standard Omniverse extension, allowing for future integration with live quantum hardware APIs.

## 📊 Scientific Validation: Stress Test Results
The simulation results validate a clear inverse correlation between physical stress and quantum stability. As the "Environmental Energy" (Fall Height) increases, the error rate (Beta) scales accordingly:

| **5.0m (Baseline)** | 100% | 0% |
| **11.0m (Mid-Stress)** | 60% | 40% |
| **17.0m (High-Stress)** | 20% | 80% |
| **23.0m (Extreme)** | 10% | 90% |

## 📂 Project Structure
* `ext/quantum_omni_tool/`: Core extension logic and USD stage management.
* `run_extension_test.py`: Entry point for the simulation and data acquisition pipeline.
* `quantum_stress_results.csv`: Empirical dataset generated by the latest simulation run.
* `extension_output.usda`: The universal scene description file for 3D visualization in Omniverse.
